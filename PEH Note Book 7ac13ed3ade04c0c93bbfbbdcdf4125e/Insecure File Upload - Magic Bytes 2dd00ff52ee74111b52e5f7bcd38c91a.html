Insecure File Upload - Magic Bytes

To solve this challenge we need to understand what this website is actually doing.

We can only upload image file. To exploit we will change the file data and extension the burpsuite.

Once we upload a file we can see the response in the http history tab in burpsuite.

Now we have to change the file data and extension.

We can see it says “Only '.jpg' and '.png' files are allowed. Sorry, your file was not uploaded.”

That means it is validating the file , Now we need to check how it validating the file either with the file extension or with the magic bytes .

  • For file extension we search on google for something like valid php file extensions.
  • For the magic bytes we need to change little bit of data.

Firstly with Magic bytes:

To see magic bytes of an image :

We can see data of the image, There is “PNG”

So we can just add our php code in between the magic bytes.

That means the validation is only for the magic bytes. Here we can also change the file extension as discussed above.

Now all we need to do is execute this file.

We have compromised the website.

Fix for this vulnerability :

Never block certain extensions always say only these extensions are allowed everything else will not be entertained.
